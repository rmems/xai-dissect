## Research question

Can static analysis of Grok-1 ckpt-0 produce a deterministic, layer-consistent map of expert tensors, router tensors, routing-critical regions, and candidate SAAQ target tensors without running inference or mutating weights?

## Primary hypothesis

Static structural analysis of Grok-1 ckpt-0 is sufficient to produce a reliable SAAQ-readiness manifest for downstream experimentation.

## Null hypothesis

Static analysis is insufficient because the checkpoint layout, quantization pairing, routing geometry, or tensor metadata are too ambiguous to produce a reliable manifest without manual interpretation or runtime execution.

## Predictions

### P1: Inventory determinism

`xai-dissect inventory` will parse the checkpoint and emit deterministic tensor records with valid dtype, shape, offset, and byte-length accounting.

Pass criteria:
- No parser panic.
- No unexplained malformed tensors.
- Repeated run produces equivalent inventory output.

### P2: Expert atlas consistency

`xai-dissect experts` will identify a regular MoE expert structure across layers.

Pass criteria:
- Expert projections can be separated from router, norm, embedding, and attention families.
- Layer-wise expert geometry is consistent enough to summarize.

### P3: Router criticality isolation

`xai-dissect routing-report` will identify router/gate tensors and flag routing-critical regions.

Pass criteria:
- Router/gate tensors are distinguishable by shape/signature.
- Routing tensors are not mixed with candidate compression/SAAQ targets.

### P4: Static stats usefulness

`xai-dissect stats` will produce tensor-family statistics useful for ranking candidate SAAQ targets.

Pass criteria:
- Stats report shows enough variance/outlier/norm information to distinguish safer target families from protected ones.
- Sampling is stable enough that rankings are not obviously noise-driven.

### P5: SAAQ-readiness manifest

`xai-dissect saaq-readiness` will emit a candidate manifest usable by Corinth-canal/SAAQ tooling without hand-editing tensor identities.

Pass criteria:
- Candidate manifest includes tensor identity, shard provenance, dtype, shape, role, and ranking/risk metadata.
- Protected routing/norm regions are excluded or explicitly flagged.
- Downstream integration path is clear.

## Falsifiers

This hypothesis fails if:
- The parser cannot process ckpt-0 without manual shard exclusion.
- Tensor shapes, offsets, or byte lengths are inconsistent.
- Expert and router families cannot be separated.
- Candidate manifests include obvious protected tensors without warning.
- Downstream use requires manual tensor-name surgery.

This hypothesis is inconclusive if:
- The CLI runs but classification confidence is too low.
- Schema fields are missing but the underlying checkpoint structure appears recoverable.
- The tool needs another parser/export iteration before the hypothesis can be judged.
